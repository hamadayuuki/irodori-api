# User Insight with Fashion Review Integration

## 概要

ユーザーインサイト生成時に、**実際のコーディネート履歴（fashion-review）**も活用するように機能を拡張しました。

---

## 🎯 実装内容

### 1. **Fashion Review データの活用**

インサイト生成時に以下のデータを統合：
1. ファッションタイプ診断結果
2. 動物占い結果
3. **実際のコーディネート履歴（過去7件）** ✨ NEW

### 2. **重み付け戦略**

- **直近3件**: 最重要として扱う（ユーザーの現在のスタイル傾向）
- **4-7件目**: 参考情報として扱う（過去のスタイル傾向）

#### 1. **新規メソッド追加**: `get_recent_fashion_reviews()`

```python
    """
    ユーザーの最近のファッションレビュー（コーディネート）を取得

    Returns:

                "coordinate_id": "...",

                "ai_review_comment": "モノトーンでまとめた...",
                "tags": ["カジュアル", "モノトーン", ...],

                "created_at": TIMESTAMP

            ...
        ]
    """
```

#### 2. **`generate_insight()` の修正**

```python
# 最近のファッションレビュー（過去7件）を取得
fashion_reviews = self.get_recent_fashion_reviews(user_id, limit=7)

# Geminiプロンプト構築（fashion_reviewsを追加）
prompt = self._build_insight_prompt(fashion_type, animal_fortune, fashion_reviews)
```

## 🚀 API エンドポイント

### インサイト生成（fashion-review統合版）

**エンドポイント**: `GET /api/user-insight?userid={user_id}`

**動作**:
1. ファッションタイプ診断結果を取得
2. 動物占い結果を取得
3. **過去7件のfashion-reviewを取得** ✨
4. Geminiで統合インサイトを生成（直近3件を最重視）
5. Firestoreに保存して返す

---

## 💡 利用シーン

### Before（診断結果のみ）
- 診断結果に基づいた一般的なアドバイス
- タイプ別の抽象的なおすすめスタイル

### After（実際のコーデ履歴活用）✨
- **実際のコーデ傾向を踏まえた具体的なアドバイス**
- ユーザーの「強み」を実データから分析
- より個別化されたスタイル提案
- コーデの改善点や新しい挑戦の提案

---

## 📋 データ取得優先順位

1. **ファッションタイプ**: 必須（なければ生成不可）
2. **動物占い**: 必須（なければ生成不可）
3. **Fashion Review**: オプション（0件でもインサイト生成可能）

**Fashion Review が 0件の場合**:
- 従来通り、ファッションタイプと動物占いのみでインサイト生成
- プロンプトにfashion-review情報は含まれない

**Fashion Review が 1-6件の場合**:
- 存在する全てのコーデ情報を活用
- 直近3件（または存在する全件）を重視

**Fashion Review が 7件以上の場合**:
- 最新7件のみ取得
- 直近3件を最重視、4-7件目を参考情報

---

## ⚙️ パフォーマンス考慮

- Fashion Reviewの取得はFirestoreインデックス不要（Pythonでソート）
- 最大7件のみ取得（プロンプトサイズを制限）
- レビューコメントは100文字まで切り詰め（プロンプトサイズ削減）
- タグは最大5個まで（直近3件）、3個まで（4-7件目）
