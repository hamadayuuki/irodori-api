# Firestore スタンダードアイテム重複削除手順書

## 🎯 目的

Firestoreの`items`コレクションに存在する重複したスタンダードアイテムを削除します。

## ⚠️ 重要な注意事項

1. **必ずバックアップを取ってください**
   - Firestoreコンソールでデータをエクスポート
   - または、この処理の前に手動バックアップ

2. **Dry-runで確認してから実行してください**
   - 最初は必ず`--dry-run`で実行
   - 削除対象を確認してから`--execute`

3. **削除は取り消せません**
   - 実行後は元に戻せません

## 📋 前提条件

- Python 3.8以上
- Firebase Admin SDKがセットアップ済み
- Firestoreへの書き込み権限

## 🔍 処理の仕組み

### 重複の判定基準

**`storage_url`が同じ**アイテムを重複とみなします。

### 残すアイテムの選択

各重複グループから**最も古い`uploaded_at`のアイテム**を1つ残します。

例:
```
同じ storage_url を持つアイテム:
  - abc123 (uploaded: 2026-03-10T10:00:00Z) ← これを残す（最古）
  - def456 (uploaded: 2026-03-12T14:00:00Z) ← 削除
  - ghi789 (uploaded: 2026-03-15T08:00:00Z) ← 削除
```

### 除外されるアイテム

以下のアイテムは処理対象外です：
- `storage_path`が`standard-items/`で始まらないアイテム
- `is_standard`が`False`のアイテム

## 🛡️ 安全対策

### 1. Dry-runモードがデフォルト

```bash
# これは安全（削除なし）
python cleanup_duplicate_items.py --dry-run
```

### 2. 実行時の確認プロンプト

`--execute`でも、実行前に確認を求めます。

### 3. バッチ処理

500件ずつのバッチで削除するため、大量のアイテムでも安全です。

### 4. エラーハンドリング

削除中にエラーが発生しても、処理済みのバッチは影響を受けません。

## 🐛 トラブルシューティング

### エラー: `FileNotFoundError: Prompt file not found`

**原因**: `prompt_loader.py`が見つからない

**解決策**: スクリプトを`irodori-api`ディレクトリで実行してください
```bash
cd /path/to/irodori-api
python cleanup_duplicate_items.py --dry-run
```

### エラー: `Permission denied`

**原因**: Firestoreへの書き込み権限がない

**解決策**:
1. Firebase認証情報を確認
2. サービスアカウントに適切な権限があるか確認

### 処理が途中で止まる

**原因**: ネットワークエラーまたはFirestore制限

**解決策**:
1. `--gender`オプションで分割処理
2. インターネット接続を確認
